MacroAPI (2.0.0)

https://docs.macrodigital.ru/manual/integracii/api-2.0

All API v2 endpoints are called via POST only with Content-Type: application/json. Action names like getUsers, list, statuses, getRecord are business action names in the URL and are not HTTP method hints.

Global rate limit for API v2 is applied per API key:

  • 100 requests per minute
  • algorithm: sliding_window
  • excess requests return 429 with X-RateLimit-* headers

EstateBuy

Создание заявки на покупку недвижимости

Создаёт лид (заявку) на покупку недвижимости. Контакт клиента ищется по phone/email в текущей и партнёрских компаниях; если не найден — создаётся новый (отдельный contacts/create перед этим вызовом не нужен). Размещение платежа в графике сделки

Проводит фактический платёж по сделке (dealId — shows_id сделки после estateDeals/changeStatus → deal). Идемпотентен по externalId: повторный вызов с тем же ID обновит существующий платёж, а не создаст дубль — подходит для безопасного ретрая импорта. Итог размещения виден в estateDeals/listPaymentSchedule.

Стратегия размещения (положительная сумма) — первый подходящий вариант:

  1. Ищется совпадение по сумме среди неоплаченных плановых платежей с ближайшей датой.
  2. Ищется любая комбинация неоплаченных плановых платежей на одном счёте, сумма которых точно равна входящей; первый из набора проводится, остальные удаляются.
  3. Если входящая меньше первого планового — он уменьшается до входящей и проводится; разница создаётся новым плановым платежом.
  4. Если больше — первый плановый увеличивается до входящей и проводится; следующие плановые поглощаются, пока разница не закроется.
  5. Если неоплаченных плановых нет, но ранее уже были импортированные — создаётся новый платёж по их шаблону. Иначе платёж не размещается.

Возвраты (отрицательная summa): создаётся возвратный платёж; при options.createCompensation: true дополнительно создаётся компенсирующий плановый платёж, если возврат нарушил баланс графика. Нулевая сумма запрещена.
