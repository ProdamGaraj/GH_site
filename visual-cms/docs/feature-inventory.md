# Visual CMS — Инвентаризация функционала

> Дата исходной инвентаризации: 6 апреля 2026
> Актуализировано: 19 мая 2026
> Проект: Visual CMS (no-code/low-code конструктор сайтов)
> Стек: React 18 + TypeScript + Redux Toolkit / Node.js + Express + TypeORM + PostgreSQL + Redis

Этот документ — канонический реестр функций. Текущее техническое состояние и
техдолг — в [../PROJECT_STATUS_REPORT.md](../PROJECT_STATUS_REPORT.md).

## Актуализация статусов (19 мая 2026)

Следующие пункты части 2 переопределяются (имеет приоритет над таблицами ниже):

- C1 «Аутентификация», C2 «RBAC» — выносятся во внешний сервис авторизации
  (микросервисная авторизация, гранулярные роли) по решению владельца.
  Вне скоупа данного репозитория; в коде не реализуются.
- C6 «Миграции БД» — реализовано: `synchronize: false`, идемпотентные SQL-миграции
  применяются на старте через `runSafeMigrations` (`backend/src/server.ts`).
- H1 «Загрузка медиа» — реализовано: модель `MediaAsset`, `MinioStorageService`,
  обработка через `sharp`, маршруты `/api/media`.
- C5 «E2E тесты» — не реализовано (E2E отсутствуют).
- H8 «Расширение тестов» — частично: добавлен `backend/jest.config.js` (ts-jest),
  backend-тесты запускаются (21 suite / 447 тестов), frontend Vitest (176).
  Покрытие как метрика не измеряется.
- 11.7 «Health Checks /metrics, readiness» и 11.9 «Prometheus метрики» —
  РЕАЛИЗОВАНО (A1 закрыт 2026-05-19): `routes/health.ts` смонтирован в `app.ts`,
  доступны `/health`, `/health/live`, `/health/ready`, `/health/detailed`,
  `/metrics`. См. [../KNOWN_ISSUES.md](../KNOWN_ISSUES.md), A1.

Состояние недоработок и нестабильных мест отслеживается в
[../KNOWN_ISSUES.md](../KNOWN_ISSUES.md). Остальные пункты ниже актуальны на
дату исходной инвентаризации.

### Критичность: 🔴 Критическая (блокер для продакшена)

| # | Функция | Описание | Обоснование |
|---|---------|----------|-------------|
| C1 | **Аутентификация и авторизация** | Система логина / регистрации / ролей / JWT-токенов. Все API-эндпоинты сейчас публичные. | Без auth невозможен запуск в продакшен — любой может редактировать/удалять контент |

| C4 | **HTTPS для бэкенда** | SSL-терминация на уровне backend или через nginx reverse-proxy для всех API | Mixed content issues между https://frontend и http://backend — текущий workaround через nginx proxy |
| C5 | **E2E тесты** | Playwright/Cypress тесты для критических сценариев: создание страницы → привязка данных → публикация → проверка | Нет ни одного E2E теста — регрессии находятся только вручную |
